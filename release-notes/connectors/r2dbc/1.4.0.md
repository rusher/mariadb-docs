# MariaDB Connector/R2DBC 1.4.0 Release Notes

## Overview

MariaDB Connector/R2DBC is a non-blocking interface between Java applications and MariaDB Server. MariaDB Connector/R2DBC enables the development of Java 8+ applications for MariaDB database products.

**Release date:** 10 Feb 2026

MariaDB Connector/R2DBC 1.4.0 is a [_**Stable**_](../../../community-server/about/release-criteria.md) _**(GA)**_ release.


## Notable Changes

* [R2DBC-109](https://jira.mariadb.org/browse/R2DBC-109) Add `fallbackToSystemTrustStore` and `fallbackToSystemKeyStore` options. see [Options](https://mariadb.com/docs/connectors/mariadb-connector-r2dbc/mariadb-connector-r2dbc-connection-parameters#connection-parameters)
* [R2DBC-110](https://jira.mariadb.org/browse/R2DBC-110) Support java.time.Instant parameters
* [R2DBC-114](https://jira.mariadb.org/browse/R2DBC-114) Implement Wrapped interface to expose EventLoop scheduler for r2dbc-pool optimization

## Bugs Fixed

* [R2DBC-108](https://jira.mariadb.org/browse/R2DBC-108) Handle authentication plugin multi-exchange prefix (0x01) introduced in MDEV-37554
* [R2DBC-111](https://jira.mariadb.org/browse/R2DBC-111) Potential hang when upstream subscription is cancelled before demand
* [R2DBC-112](https://jira.mariadb.org/browse/R2DBC-112) Failed authentication when using `caching_sha2_password` with passwords longer than 18 characters
* [R2DBC-113](https://jira.mariadb.org/browse/R2DBC-113) Add support for RSA public key content in cachingRsaPublicKey and rsaPublicKey options
